# Чеклист смены сезона (N → N+1)

Код бэкенда и приложения переезжает на новый сезон сам: `YEAR` всех продьюсеров
флипается 1 января по UTC, season-guard'ы (`src/season.ts`) закрывают окна
рассинхрона с Jolpica/FIA, WEC/f1-зеркала подхватывают новые сезонные страницы
по мере публикации. Ниже — то, что автоматизации не поддаётся: курируемые
данные. Даты — по традиционному календарю публикаций.

## Август–сентябрь (публикация календарей)

- [ ] **`SCHEDULE[N+1]` в `src/schedule.ts`** — календарь IMSA публикуют в
  августе (после Road America). Это единственный жёсткий блокер: без него
  продьюсер imsa падает с 1 января ежечасно (fail-loud задуман, но письма
  каждый час). Round/name/venue/даты; venue должен матчиться с будущими
  папками Al Kamel (`matchTrack`).
- [ ] Проверить, что fiawec опубликовал `/en/season/N+1` (обычно к осени уже
  есть; в 2026 страница 2027 была live уже в июле) — тогда WEC-межсезонье
  пройдёт гладко само.

## Осень (составы и медиа — репо overlap-assets и приложение)

- [ ] Медиа событий: папки `images/{f1,wec,imsa}/events/<YY>/` с роликом
  `<round>.MP4` и постером `<round>-.jpg` на каждый этап (иначе карточки —
  серые заглушки). Заливать можно по мере анонсов.
- [ ] Приложение, карты составов (протухают тихо):
  - `WECManufacturer.carNumbers` — **самая опасная**: устаревший номер отдаёт
    УВЕРЕННО неверную марку (не пропуск!). Сверить весь грид.
  - `WECStanding.driverFlags` — новые пилоты без записи = пустой вырез флага.
  - `TeamColors` / `F1TeamSlug` — новые команды/ребрендинги F1.
  - `F1CircuitFacts` — спринт-сетка нового сезона (lapsSprint!), компаунды,
    счётчики «по N-1», contractUntil; `SeriesEventFacts` — счётчики изданий.
  - Новые трассы: координаты (`WeatherService`), контур (assets + slugOverride),
    таймзона IMSA (`trackTimeZone`), `TrackFacts`.
  - Портреты новых пилотов и лого команд в overlap-assets.

## Декабрь (после финалов)

- [ ] Разовый локальный прогон `SEASON=N npm run openf1` — добрать дыры
  зеркала уходящего сезона (после флипа YEAR его раунды недостижимы для
  backfillPit).
- [ ] Убедиться, что финальные year-копии на месте:
  `data/f1/jolpica/N.json`, `N_driverStandings.json`,
  `N_constructorStandings.json` (пишутся автоматически из current-алиасов).

## Январь (флип)

- [ ] Логи первых прогонов: guard-сообщения «переходное окно, пропускаем» у
  fia/winners/milestones — норма, пока Jolpica не зальёт сезон N+1.
- [ ] Когда FIA создаст ноду сезона — обновить `SEASON_URL_FALLBACK` в
  `src/fia.ts` (до тех пор при провале авто-дискавери прогон честно
  пропускается, не скрейпит чужой год).
- [ ] После первых гонок: спот-чек `data/f1/fia/N+1_1.json` (carryOver из
  финала N, если был штраф «на следующую гонку»), winners/milestones файлы
  нового сезона.
